# BuildIt 2025 - Hackathon Submissions

## 🚀 About BuildIt 2025

**BuildIt 2025** is a **12-hour hackathon** organized by the Computer Science Engineering Club to bring together talented students to solve real-world problems using **AI/ML Solutions for Social Good**.

### Event Details
- **Theme:** AI/ML Solutions for Social Good
- **Duration:** 12 hours (9 AM - 9 PM) or 24 hours
- **Team Size:** 2-4 participants per team
- **Expected Teams:** 10-20 teams
- **Submission:** GitHub Repositories
- **Judging:** Manual evaluation by faculty and industry experts

---

## 📋 Problem Statements

Teams can choose to solve one of the following problems:

### Problem 1: AI Chatbot for Student Mental Health
Develop an intelligent chatbot that provides mental health support and resources to students. The solution should:
- Detect emotional distress indicators
- Provide empathetic responses and coping strategies
- Connect users with professional resources when needed

### Problem 2: ML Model to Predict Crop Yield
Create a machine learning model that predicts crop yield based on environmental and farming data. The solution should:
- Analyze historical crop data
- Incorporate weather patterns and soil conditions
- Provide actionable recommendations to farmers

### Problem 3: Plagiarism Detection App
Build an application that detects plagiarism in student assignments. The solution should:
- Compare submitted work with existing sources
- Identify similarities and potential plagiarism
- Generate detailed plagiarism reports

### Problem 4: Open Innovation (Optional)
Propose and build your own AI/ML solution for social good that aligns with the hackathon theme.

## 📝 Submission Guidelines

### Repository Structure
Each team must create a GitHub repository using the template provided and include:

1. **README.md** with complete project documentation
2. **Source Code** in organized folders (frontend, backend, models)
3. **Documentation** including setup instructions
4. **Screenshots/Demo Video** showcasing the solution

### Required Fields in README
- Project title and team name
- Problem statement selection
- Team member details (name, roll number, email)
- Project description and problem it solves
- Technology stack used
- Setup and run instructions
- Screenshots (3-5 images)
- Demo video link (YouTube or deployed link)
- Challenges faced
- Future improvements

### Submission Process
1. Fork or create a new repository from the team template
2. Complete all project files and documentation
3. Push all code to GitHub before submission deadline
4. Fill out the **Google Form** with:
   - Team name and members
   - GitHub repository link
   - Demo link (if available)
   - Tech stack used

---

## ⭐ Judging Criteria

Projects will be evaluated based on:

1. **Innovation (25%):** Uniqueness and creativity of the solution
2. **Impact (25%):** Real-world applicability and social benefit
3. **Technical Quality (25%):** Code quality, architecture, and implementation
4. **Presentation (25%):** Documentation, demo quality, and clarity
